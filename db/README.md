| File | Purpose |
|---|---|
| `pragya_yoga_schema.sql` | Creates all 5 tables, primary/foreign keys, constraints, and indexes. Run this first. |
| `pragya_yoga_data_first20.sql` | Populates the schema with the first 20 asanas (source: Master Asana Database Workbook v1.1.1). Run this second. |

## How to set this up locally

**PostgreSQL:**
```bash
createdb pragya_yoga_verse
psql -d pragya_yoga_verse -f pragya_yoga_schema.sql
psql -d pragya_yoga_verse -f pragya_yoga_data_first20.sql
```

**MySQL:**
```bash
mysql -u root -p -e "CREATE DATABASE pragya_yoga_verse;"
mysql -u root -p pragya_yoga_verse < pragya_yoga_schema.sql
mysql -u root -p pragya_yoga_verse < pragya_yoga_data_first20.sql
```

> Note for PostgreSQL: the schema uses `AUTO_INCREMENT` (MySQL syntax) for
> surrogate keys. On PostgreSQL, translate this to
> `GENERATED BY DEFAULT AS IDENTITY` (not `GENERATED ALWAYS`) so the data
> script can insert explicit ID values for categories 1–10. This was
> confirmed by actually running both scripts against a live PostgreSQL 16
> instance during this task.

## Verification performed

The scripts were executed end-to-end against a live database (not just
checked by reading the SQL). Confirmed:

- **Row counts match spec exactly**: 6 Categories, 20 Asanas, 100 Benefits, 80 Precautions, 20 Chakra_Information records.
- **Zero orphaned rows** — every `asana_id` in Benefits, Precautions, and Chakra_Information resolves to a real row in Asanas; every `category_id` in Asanas resolves to a real row in Categories.
- **Joins resolve correctly** — spot-checked by joining Asanas → Categories and counting each asana's Benefits/Precautions.

Full query output is in `verification_output.txt`.
